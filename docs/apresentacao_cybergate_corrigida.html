CyberGate

Sistema ciberfisico de controle de acesso com ESP32

Sistema embarcado com deteccao ultrassonica, autenticacao RFID, servo motor, interface web e monitoramento de performance.

Performance em Sistemas Ciberfisicos - PUCPR Andre Gritten, Mateus Furtado, Joao Mezari e Gustavo Isdra
Objetivo

Implementar e validar um controle de acesso fisico com observabilidade embarcada

O projeto demonstra a integracao entre sensores, atuador, concorrencia em FreeRTOS, interface web e persistencia local para analisar comportamento e performance do sistema.

Escopo implementado

Funcionalidades reais do firmware

  • Deteccao de veiculo por sensor ultrassonico HC-SR04.
  • Autenticacao por RC522/MFRC522 com UID autorizado no firmware.
  • Abertura e fechamento por servo motor com auto-fechamento.
  • Interface web embarcada com status, controle, logs, configuracao e performance.
  • Persistencia local com LittleFS e configuracoes com Preferences.
Hardware

Componentes utilizados

ESP32

Microcontrolador dual core com Wi-Fi integrado e suporte a FreeRTOS.

HC-SR04

Sensor ultrassonico usado para confirmar a presenca do veiculo.

RC522/MFRC522

Leitor RFID usado na autenticacao da tag.

Servo motor

Atuador do portao, com posicoes aberta e fechada calibraveis.

Arquitetura

Organizacao real das tasks

  • SensorTask no core 1: ultrassonico, RFID e decisao de abertura.
  • ControlTask no core 0: servo, auto-fechamento, metricas e persistencia periodica.
  • AsyncWebServer: paginas estaticas e endpoints REST.
  • loop(): monitoramento do Wi-Fi e entrada em Light Sleep quando ocioso.

Arquivos como rfid_task.cpp, web_task.cpp e log_task.cpp estao vazios e nao representam tasks ativas.

Fluxo operacional

Da deteccao a abertura do portao

  1. O ultrassonico mede distancia com interrupcao, timeout e mediana de amostras.
  2. Com veiculo confirmado, o RFID e ativado.
  3. A tag lida e comparada com a lista autorizada do firmware.
  4. Se autorizada, o portao abre e o evento e registrado.
  5. Depois do intervalo configurado, o portao fecha automaticamente.
Interface web

Dashboard embarcado no ESP32

  • Paginas hospedadas no LittleFS.
  • Status em tempo real por consultas periodicas.
  • Controle remoto do portao e calibracao do servo.
  • Logs, exportacao CSV, configuracao operacional e Wi-Fi.
  • Performance com heap, CPU estimada, rede, sensores, tasks e historicos.

A comunicacao atual e API REST com polling, nao WebSocket.

Observabilidade

Metricas e persistencia

  • Eventos de sensor, RFID, controle, usuario, rede, configuracao e erros.
  • Heap livre, heap minimo, maior bloco alocavel, PSRAM e uso de LittleFS.
  • Tempo ultimo, medio e maximo das funcoes instrumentadas.
  • Stack livre das tasks principais.
  • Logs e amostras de metricas persistidos no LittleFS com rotacao por tamanho.
Rede e energia

Conectividade e baixo consumo

  • O ESP32 tenta conectar ao Wi-Fi salvo.
  • Em falha ou ausencia de credenciais, abre AP de configuracao CyberGate-Setup.
  • Light Sleep e usado quando o sistema esta ocioso.
  • A entrada em Light Sleep e bloqueada durante deteccao, abertura, calibracao, movimento do servo ou acesso web recente.
  • O despertar atual e feito por timer, nao por interrupcao externa de GPIO.
Validacao

Testes coerentes com o projeto

  • Teste isolado do ultrassonico, RFID e servo.
  • Fluxo integrado: detectar veiculo, validar tag e abrir portao.
  • Monitoramento via dashboard durante execucao.
  • Exportacao de logs para evidenciar comportamento.
  • Resultados como 24 horas de execucao devem ser apresentados somente com logs ou medicoes coletadas.
Limitacoes atuais

Pontos importantes para nao superestimar a implementacao

  • UIDs autorizados ficam fixos no firmware.
  • Nao ha banco de dados externo.
  • Nao ha WebSocket implementado.
  • A estimativa de CPU e aproximada.
  • Alguns arquivos de services/tasks estao reservados para evolucao futura, mas ainda vazios.
Conclusao

CyberGate demonstra controle fisico, concorrencia e observabilidade em baixo custo

O sistema implementa um prototipo funcional com ESP32, FreeRTOS, sensor ultrassonico, RFID, servo motor, interface web embarcada, API REST, persistencia local e metricas de desempenho. A apresentacao corrigida descreve exatamente essa arquitetura.
